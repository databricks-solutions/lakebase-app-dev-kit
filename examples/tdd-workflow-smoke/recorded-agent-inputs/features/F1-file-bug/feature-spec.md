# File a Bug in the Browser

## Summary

Team members can open the app in a browser, fill in a title and description on a create form, and submit to file a bug. The bug is created with an `open` status and is accessible via a stable, shareable URL (e.g., `/bugs/42`) that displays the bug's title, description, status, and identifier.

## Stories

- S1-create-bug: Fill in a title and description on a form and submit it
- S2-view-bug-detail: View a bug's details at a stable URL

## Out of scope

State management by the team (states are fixed as `open` for now), list or queue views, editing or deleting bugs after creation, assigning or reassigning owners, search or filtering, and any other capabilities deferred to later sprints.
