# S1 - File a bug

**As a** team member  
**I want to** fill in a form with a bug's title and description and submit it  
**So that** I can create a bug in the tracker

## Summary

The user interacts with a create-bug form in the browser, enters a bug's title and description, and submits the form to create the bug. The newly created bug starts in the `open` state and is assigned a unique identifier. On successful submission, the user is navigated to the bug's detail page.

## E2E scope

This is an end-to-end (UI) story: the user directly interacts with the browser form to file the bug.
